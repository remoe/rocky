Detailed Description

Base class for all Map layers.

Subclass Layer to create a new layer type. Use the META_Layer macro to establish the standard options framework.

When you create a Layer, init() is called. Do all one-time construction activity where.

When you add a Layer to a Map, the follow methods are called in order:

setReadOptions() sets OSG options for IO activity; open() to initialize any underlying data sources; addedToMap() to signal to the layer that it is now a member of a Map.

How (and if) to use this layer when rendering terrain tiles.

Enumerator
RENDERTYPE_NONE 

Layer does not draw anything (directly)

RENDERTYPE_TERRAIN_SURFACE 

Layer requires a terrain rendering pass that draws terrain tiles with texturing.

RENDERTYPE_TERRAIN_PATCH 

Layer requires a terrain rendering pass that emits terrian patches or invokes a custom drawing function

RENDERTYPE_CUSTOM 

Layer that renders its own node graph or other geometry (no terrain)
